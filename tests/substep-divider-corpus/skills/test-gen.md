# Skill: test-gen

**Shape:** iterates over each module in modules.

**Distinct work-units this skill performs** (each is a phase the user will look for on the board):

- **analyze module** — analyze, read module.
- **write tests** — write tests, add cases.
- **edge cases** — edge case, boundary, error path.  _(visibility divider — no hard gate needed)_
- **run suite** — run suite, coverage.

## Procedure
For each module in modules:
  1. analyze module — analyze, read module.
  2. write tests — write tests, add cases.
  3. edge cases — edge case, boundary, error path.
  4. run suite — run suite, coverage.
